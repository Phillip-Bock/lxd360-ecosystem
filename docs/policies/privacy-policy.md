## 1. Purpose

This policy establishes how LXD360 collects, processes, stores, and protects personal data in compliance with applicable privacy regulations including GDPR and CCPA.

## 2. Scope

This policy applies to:
- All personal data processed by LXD360
- All employees handling personal data
- All systems processing personal data
- All third parties with access to personal data

## 3. Definitions

| Term | Definition |
|------|------------|
| **Personal Data** | Information relating to an identified or identifiable person |
| **Processing** | Any operation performed on personal data |
| **Data Subject** | Individual whose personal data is processed |
| **Controller** | Entity determining purposes and means of processing |
| **Processor** | Entity processing data on behalf of controller |
| **DPO** | Data Protection Officer |

| Category | Examples | Legal Basis |

## 5. Data Subject Rights

### 5.1 GDPR Rights (EU/EEA)

| Right | Description | Response Time |
|-------|-------------|---------------|
| Access (Art. 15) | Obtain copy of personal data | 30 days |
| Rectification (Art. 16) | Correct inaccurate data | 30 days |
| Erasure (Art. 17) | Request deletion ("right to be forgotten") | 30 days |
| Restriction (Art. 18) | Limit processing | 30 days |
| Portability (Art. 20) | Receive data in machine-readable format | 30 days |
| Object (Art. 21) | Object to processing | 30 days |
| Automated Decision (Art. 22) | Challenge automated decisions | 30 days |

### 5.2 CCPA Rights (California)

| Right | Description |
|-------|-------------|
| Know | Know what data is collected |
| Delete | Request deletion |
| Opt-out | Opt-out of sale of data |
| Non-discrimination | No retaliation for exercising rights |

### 5.3 Exercising Rights

Data subjects may exercise rights by:
- Email: privacy@lxd360.com
- Platform: Account settings > Privacy
- Mail: [Company address]

## 6. Data Processing

### 6.1 Processing Principles

| Principle | Implementation |
|-----------|----------------|
| Lawfulness | Valid legal basis documented |
| Fairness | Transparent processing |
| Transparency | Clear privacy notices |
| Purpose limitation | Data used only for stated purposes |
| Data minimization | Collect only necessary data |
| Accuracy | Mechanisms to update data |
| Storage limitation | Defined retention periods |
| Integrity | Security controls |
| Accountability | Documentation and audits |

### 6.2 Data Minimization

- Collect only data necessary for stated purpose
- Regularly review data holdings
- Delete data no longer needed

### 6.3 Purpose Limitation

Data collected for specified purposes shall not be processed for incompatible purposes without:
- Additional consent, or
- Legal basis, or
- Compatible legitimate interest

## 7. Data Retention

### 7.1 Retention Periods

| Data Category | Retention Period | Justification |
|---------------|------------------|---------------|
| Account data | Duration of account + 3 years | Contract + legal |
| Learning records | 7 years | Regulatory compliance |
| xAPI statements | 10 years | EU AI Act requirement |
| Payment records | 7 years | Tax/legal requirements |
| Audit logs | 7 years | Compliance |
| Marketing data | Until consent withdrawn | Consent-based |
| Analytics data | 26 months (anonymized) | Legitimate interest |

### 7.2 Deletion Procedures

Upon retention period expiry or deletion request:
1. Verify identity of requester
2. Identify all data locations
3. Execute secure deletion
4. Document deletion
5. Notify processors to delete

## 12. Data Breach Response

### 12.1 Breach Notification

**To Supervisory Authority (GDPR Article 33):**
- Within 72 hours of awareness
- Unless unlikely to result in risk

**To Data Subjects (GDPR Article 34):**
- Without undue delay
- If high risk to rights and freedoms

### 12.2 Breach Documentation

All breaches documented with:
- Facts of breach
- Effects
- Remedial action taken
